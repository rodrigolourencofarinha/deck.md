# Single slide workflow

Use this note to build or revise one slide.

For consulting-style slides, also use `consulting-slide-standards.md` as the quality bar.

## Core rule

Build in two passes:
1. logic pass
2. polish pass

Do not polish early if the claim and proof are still weak.

Consulting rule:
- one slide, one insight
- title states the insight
- body proves the title
- footer shows the receipt when data, dates, or caveats matter, with the standard small `CR` mark and simple page number unless disabled

## Inputs

Start with:
- one clear claim
- the evidence that supports it
- the audience
- the slide type
- a template, prior slide, or archetype to reuse when helpful

Do not start from decoration.

## Build order

1. Write the slide's working message as a plain sentence.
2. Choose the slide type that best proves that message.
3. Sketch the main slide zones.
4. Put a working action title on the slide early.
5. Build the body skeleton first.
6. Insert the raw proof.
7. Delete anything that does not help prove the claim.
8. Add subheadings, callouts, and emphasis.
9. Tighten the title so it matches the body exactly.
10. Add source, units, dates, notes, and qualifiers.
11. Align, standardize, and simplify.
12. Read the rendered slide like a busy executive.

Working rule:
1. make it true
2. make it clear
3. make it pretty

## Common patterns

### Chart plus interpretation
Use for trends, comparisons, benchmarks, or before/after logic.

Typical shape:
- action title at top
- subheading states what, unit, scope, and period
- chart as primary proof, using the simplest chart type that carries the insight
- short takeaway bullets or callouts beside or below it
- source and notes in the footer, kept separate from the standard `CR` mark and page number

### Summary or conclusion page
Use for executive summaries, section summaries, and recap slides.

Typical shape:
- strong title
- grouped bullets or modules
- bolded takeaway phrases
- optional summary banner only when it adds value

### Modular proof blocks
Use for 2 to 4 supporting arguments under one central claim.

Typical shape:
- one main claim
- 2 to 4 parallel evidence blocks
- simple icons or headings as anchors
- MECE grouping with no overlaps or gaps where possible

## Chart cleanup

Treat a pasted chart as raw material, not final output.

Cleanup sequence:
1. remove the chart's own default title box
2. simplify axes, labels, and gridlines
3. recolor series according to meaning
4. remove unnecessary labels and clutter
5. highlight only the numbers or series that matter
6. add the real slide title above the chart
7. add unit, date range, and source clearly
8. add a short interpretation so the reader does not infer the point alone

## Visual standards

### Layout and reading flow
- keep title position stable
- keep margins and footer behavior stable, including `CR` lower-left and the simple page number lower-right
- build an obvious reading path
- use arrows, connectors, or grouping only when they help

### Typography
- keep titles short and clean
- use one or two font families only
- keep repeated text styles consistent
- use bold for emphasis, not as the default body style

### Color
- use a restrained palette
- make the slide work in grayscale first
- reserve strong colors for emphasis
- use gray for lower-priority information

### Images and icons
- use visuals to support the argument, not fill space
- prefer simple, consistent icon style
- avoid decorative shadows and clutter
- let images earn their space

### Sources and markings
- include source lines consistently
- add notes only when needed
- mark status labels such as PUBLIC or CONFIDENTIAL consistently

## Blank-slide checklist

- what is the one point
- what evidence proves it
- what slide type fits that evidence
- does the title make a claim rather than name a topic
- does every major object support the title
- can anything be deleted
- are source, units, and dates clear
- is any number missing a unit or source
- would the title/body/footer pass the consulting-slide contract
- is the reading flow obvious
- could a busy reader get the point quickly
